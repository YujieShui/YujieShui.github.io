学会shell（二）函数

通过上一节了解了 shell 中的变量，字符串和数学运算，这一节来聊一聊 shell 中的函数。假设我们要编写一个 shell 脚本，实现加减乘除的操作，思路就会是将加减乘除的操作各自定义成一个函数，一个函数将会用输入参数来接收我们要进行运算的数字，用函数体来做相应运算，并通过返回值来返回运算结果。

所以本节目标就是写出这样一个实现加减乘除功能的脚本。

函数定义和使用

shell 中的函数有两种定义格式，使用任意一种都可以，我比较习惯用第二种形式。

我们直接来 calculate.sh 这个脚本，用一个函数，使用条件判断来实现加减传给你出的操作。

解释一下这个脚本：

  1. function calcu {…} 是 shell 函数的其中一种格式
  2. case $2 in是一个判断语句，根据$2是 +, -, *, / 来执行相应操作
  3. $1 | $2 | $3表示命令行键入的参数

使用./calculate 1 + 2来执行这个脚本

  1. 此时$1 = 1,$2 = +,$3 = 2
  2. 输出结果为 3

总结：

  1. shell 中的函数有两种定义的形式
  2. shell 中使用 $1,$2这样的形式传递参数

函数返回值

函数一般包含函数名，函数体，参数列表，返回值。上面说到 shell 函数的定义方式，参数是如何传递的，我们还需要知道 shell 中的函数是怎么返回值的。

shell 有两种返回值的方式，一种是使用return，一种是使用echo。

  • 使用return

    • 使用 return 返回值，只能返回 1-255 的整数
    • 函数使用 return 返回值。通常只能用来供其他地方调用获取状态，因此通常仅返回0或1；0表示成功，1表示失败

  • 使用echo

    • 使用 echo 可以返回任何字符串结果
    • 通常用于返回数据，比如一个字符串值或者列表值

用两个例子来体会一下

nginx.sh 采用 return 作为返回值的形式，它做了这样一件事情，使用 ps 命令并做过滤来检查系统中的 nginx 是否正常运行，将结果输出到 /dev/null 中。如果 nginx 在运行则会返回 0，反之返回 1。

之后可以用后台挂起的形式运行这个脚本，将其作为 nginx 的守护进程：nohup sh nginx.sh &，使用tail -f nohup.out来查看监听结果。

第二个脚本 get_sys_user.sh 则是使用 echo 来返回字符串，它返回的是系统中的用户名。其中定义的 get_users 这个函数会将所有用户名返回给 user_list 这个变量，并在之后做遍历操作。

局部变量和全局变量

上一节中介绍了 shell 中的变量、变量匹配和特殊变量，这一节讲到函数不得不讲的就是局部变量和全局变量，进而就讲到了变量作用域的问题。

  • 全局变量
    • 不做特殊申明，shell 中变量都是全局变量
    • tips：大型脚本程序中函数中慎用全局变量
  • 局部变量
    • 定义变量时，使用 local 关键字
    • 函数内和外若存在同名变量，则函数内部变量覆盖外部变量

运行一下var.sh体会一下即可。

编写函数库文件的建议

  • 库文件名的后缀是任意的，但一般使用 .lib
  • 库文件通常没有可执行选项
  • 库文件无序和脚本在同级目录，只需在脚本中引用时指定
  • 第一行一般使用 #/bin/echo, 输出警告信息，避免用户执行
